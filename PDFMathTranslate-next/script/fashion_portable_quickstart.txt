PDFTranslate Portable Quick Start
===========================================

Recommended startup
-------------------
Double-click Start-Fashion.bat.
It opens the FastAPI + React WebUI and preloads the fashion glossary resources
for apparel documents. If GUI login is enabled, regular users only see PDF
upload, translation, status, and download. Administrators see the Settings page
after logging in with the admin account configured in config\\distribution.toml
or PDF2ZH_* environment variables.

What each script does
---------------------
Start-Fashion.bat
  Recommended launcher.
  It starts the WebUI with no-watermark defaults, built-in fashion glossary
  support, SiliconFlowFree as the default translator, and the portable config
  folder under config\\.
  config\\distribution.toml is the simple administrator file for GUI login,
  LAN queue limits, QPS, and worker limits.
  BabelDOC models, fonts, cmaps, tiktoken cache, and generated files are kept
  under data\\ instead of the Windows user profile.
  If port 7860 is already occupied, the launcher now automatically falls back
  to the next available local port.

BABELDOC-BUILD-INFO.txt
  Records which BabelDOC source and version were packaged into this folder.

Custom terminology
------------------
config\\fashion-customer-glossary-template.csv is preloaded by default.
Administrators can edit the CSV file in place to add customer, brand, fabric
composition, BOM, and wash-care specific terms. The React Settings page also
includes a Customer Glossary Template editor for admin users.

Offline assets
--------------
The portable package now restores the required offline BabelDOC assets during
build, so you should not see a loose offline_assets_*.zip file in the final
desktop folder.
The restored assets live under data\\ and are reused after copying the whole
portable folder to another Windows machine.
